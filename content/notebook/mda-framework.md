# MDA Framework

Siehe: [[Game Design]], [[Game Studies]]

Formales Framework zur Analyse von [[Games]], das aus drei zentralen Komponenten besteht: Mechanics, Dynamics, und Aesthetics

>”**Mechanics** describe the particular components of the game, at the level of data representation and [[Algorithmen|algorithms]]. **Dynamics** describe the run-time behavior of the mechanics acting on player inputs and each others' outputs over time. **Aesthetics** describe the desirable [[Emotionen|emotional]] responses evoked in the player, when she interacts with the game system.”

\[ ⚠️ *Wie bereits von vielen angemerkt ist die Terminologie etwas irreführend, da die Begriffe im herkömmlichen/alltäglichen Gebrauch verschiedene Bedeutungen mit subtilen Unterschieden haben. “Mechanics” ist also nicht unbedingt synonym mit [[Game Mechaniken]] zu verstehen, und “Aesthetics” bezieht sich auch nicht auf [[Game Ästhetiken]] im rein audio-visuellen Sinn.* \]

Um die “Aesthetics” eines Spiels – die ja immer eine subjektive Komponente haben – besser analysieren zu können, schlägt das MDA Modell zusätzlich eine Taxonomie für “Eight Kinds of Fun” vor:

- Sensation: Game as sense-pleasure (siehe auch: [[Game Feel]])
- Fantasy: Game as make-believe (siehe auch: [[Roleplaying]], [[Fantasy]])
- Narrative : Game as drama (siehe auch: [[Narrative Design]])
- Challenge: Game as obstacle course (siehe auch: [[Puzzle]])
- Fellowship: Game as social framework
- Discovery: Game as uncharted territory (siehe auch: [[Exploration]])
- Expression: Game as self-discovery (siehe auch: [[Sandbox]])
- Submission: Game as pastime (siehe auch: [[Idle Games]])
