# Linus’s Law

So bezeichnet Eric Raymond die Grundannahme, auf der [[Linus Torvalds]]’ [[Open Source]]-Entwicklungsstil beruht:

>”Given enough eyeballs, all bugs are shallow.”

Je mehr (inoffizielle) Kollaborateure, desto wahrscheinlicher, dass alle wesentlichen Probleme identifiziert werden, und dass jemand anderes sofort eine gute Lösung kennt.
